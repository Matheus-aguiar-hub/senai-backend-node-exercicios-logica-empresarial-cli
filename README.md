# 📌 Exercícios de Lógica — Back-end com Node.js
 
## 📖 Descrição do Projeto
 
Este projeto reúne uma série de **exercícios práticos de lógica**, desenvolvidos como atividades acadêmicas no início do semestre de **Back-end**, sob orientação do professor **Marcel Teixeira**.
 
Os sistemas foram construídos com foco na **validação de dados**, **organização modular do código** e **experiência do usuário via terminal**, abordando desde cálculos matemáticos até controle de fluxo com estruturas de repetição e condicionais.
 
---
 
## 🎯 Objetivo
 
Desenvolver o raciocínio lógico aplicado ao back-end, criando sistemas funcionais com entradas do usuário, validações robustas e saídas formatadas no terminal.
 
---
 
## 📚 Exercícios
 
### Exercício 01 — IMC
Calcula o Índice de Massa Corporal do usuário com base no peso e altura informados, classificando o resultado em categorias (abaixo do peso, peso normal, sobrepeso, obesidade 1, 2 ou 3).
 
### Exercício 02 — Universidade
Sistema de gestão de notas acadêmicas. Calcula a média do aluno com base em 4 notas e determina se está aprovado, em exame ou reprovado, gerando um comprovante com os dados da simulação.
 
### Exercício 03 — Tabuada
Gera a tabuada de qualquer número informado pelo usuário, dentro de um intervalo definido, utilizando estruturas de repetição.
 
### Exercício 04 — Fatorial
Calcula o fatorial de um número inteiro positivo informado pelo usuário, com validações para entradas inválidas e casos especiais.
 
### Exercício 05 — Pares e Ímpares
Recebe um intervalo numérico e exibe todos os números pares e ímpares contidos nele, junto com a contagem de cada grupo.
 
---
 
## ⚙️ Funcionalidades
 
- Entrada e leitura de dados via terminal
- Validação de entradas do usuário em todos os exercícios
- Organização modular com separação entre lógica, cálculo e tratamento
- Exibição de resultados formatados no console
 
---
 
## 🛡️ Validações Implementadas
 
O sistema realiza verificações para garantir:
 
- Bloqueio de letras em campos numéricos
- Verificação de valores inválidos
- Tratamento de campos vazios
- Segurança dos dados
- Consistência das informações inseridas
- Funcionamento correto das regras de negócio
 
---
 
## 📝 Padronização e Boas Práticas
 
O código segue um padrão de comentários no início de cada arquivo, contendo:
 
- Objetivo do programa
- Data de criação
- Autor
- Versão do arquivo
 
Esse padrão facilita a **manutenção**, **organização** e o **entendimento do código por outros desenvolvedores**, sendo reutilizado em projetos futuros.
